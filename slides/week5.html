Software Quality Assurance

Lecture 5: xUnit Basics

What is xUnit

  • Family of Automated Test Frameworks used for automating Hand-Scripted Tests
  • Most programming languages implementations
  • Allow you to test the code using the same programming language as the source
  • Ease of access of the source code, developers can write the tests

How are tests organized

  • Tests are specified as Test Methods
  • Tests are aggregated into Test Suites that can be run as a single operation
  • Test can be automatically discovered without the need to create a test enumeration
  • Expected results are specified in the form of assertions
  • After tests are finalized a report is produced

The Four-Phase life of a Test Method

  • Set up the Test
  • Exercise public or private methods of the SUT (System Under Test)
  • Assert outputs
  • Tear Down the Test.

Fixtures

  • Everything needed to have in place to exercise the SUT
  • Databases, Filesystem, Objects, Data structures, Memory
  • Transient vs. Persistent
  • Front Door Setup: Create objects and drive them to a specific state by calling methods from its public interface(Front door setup)
  • Back Door Setup: Create objects and force them to the state by modifying its internal datastatructures (Back door setup)
  • Delegated Setup: extract fixture creation to a utility method to avoid duplication.
  • Why and when to tear down?
